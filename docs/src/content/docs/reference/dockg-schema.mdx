docmeta ships **`dockg:frontmatter:0.1`**, the frontmatter schema for
[dockg](https://github.com/hawkeyexl/dockg) — a tool that deterministically
derives an RDF knowledge graph (Turtle) from documentation frontmatter and
formatting. Documents declare SKOS concept fields under a single `kg`
frontmatter key; this schema validates that key's shape. It is not the default
schema — opt in per file, via config, or on the command line:

```bash
docmeta validate --schema dockg:frontmatter:0.1 docs/
```

| Property | Value |
|---|---|
| Id | `dockg:frontmatter:0.1` |
| Title | dockg knowledge-graph frontmatter |
| Dialect | [Draft 2020-12](/docmeta/reference/schema-resolution/#dialects) |
| Additional properties | Allowed at the root; the `kg` key itself is closed |
| Reference kind | [builtin](/docmeta/reference/schema-resolution/#reference-kinds) |

The root is open (`additionalProperties` is not restricted), so this schema
composes with other schemas — like `google:okf:0.1` — in a
[schema set](/docmeta/reference/schema-resolution/). Only the `kg` key is
constrained, and files without a `kg` key pass.

## The `kg` key

| Field | Type | Constraint |
|---|---|---|
| `prefLabel` | string | Preferred label of the concept the document is primarily about |
| `altLabels` | array of string | Unique synonyms/abbreviations; requires `prefLabel` |
| `broader` | array of string | Unique labels of broader (parent) concepts; requires `prefLabel` |
| `narrower` | array of string | Unique labels of narrower (child) concepts; requires `prefLabel` |
| `related` | array of string | Unique labels of associatively related concepts; requires `prefLabel` |
| `subjects` | array of string | Unique subject labels; behaves like tags |

The label/relation fields (`altLabels`, `broader`, `narrower`, `related`)
require `prefLabel` via `dependentRequired` — without a primary concept there
is nothing to attach them to. `subjects` stands alone.

## Example

```md
---
title: Configuration Reference
tags: [configuration]
kg:
  prefLabel: Configuration
  altLabels: [config, settings]
  broader: [Administration]
  related: [Installation]
  subjects: [reference]
---
```

dockg maps these to SKOS: `prefLabel`/`altLabels` become `skos:prefLabel`/
`skos:altLabel` on a concept node the document points at via
`foaf:primaryTopic`; `broader`/`narrower`/`related` become the corresponding
SKOS relations; `subjects` become `dcterms:subject` edges, like `tags`.
